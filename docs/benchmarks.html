How to read this

Methodology before marketing.

The benchmark suite is local and deterministic. It runs fixed corpus fixtures through AIegis guard functions, policy and MCP boundary checks, JSONL audit operations, and CLI subprocess smoke cases.

Command

python benchmarks/run.py --json

Run shape

100 iterations per benchmark. Results report min, median, p95, max, and operations per second.

Environment

Python 3.13.13 on Linux 6.17.0-23-generic, 12th Gen Intel(R) Core(TM) i7-12700H, 20 logical CPUs.

Caveats

CLI timings include Python subprocess startup. The in-process guard and decision numbers are the better signal for embedded use.

Interpretation

The useful signal is the shape of the overhead.

HTML and email are the heavier guard paths because they parse markup, MIME structure, hidden content, metadata, and attachments. Policy, tool-firewall, and MCP proxy decisions are much smaller than parser cost in this baseline. Audit appends are low-latency local JSONL writes, while audit-log verification is intentionally separated from the hot path.

These measurements are a baseline for repeatable local comparison across AIegis changes. They are not claims about other products or hosted deployments.
