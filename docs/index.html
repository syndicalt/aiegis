The problem

Agents read hostile data and then act on it.

Browsing agents, email agents, document summarizers, and autonomous tool users routinely process content that was written by someone else. That content can contain indirect prompt injection, memory poisoning, metadata traps, hidden HTML, malicious attachments, and instructions that try to steer tool use or leak data.

Indirect prompt injection

Instructions embedded in web pages, email bodies, PDFs, CSS, or metadata.

Data and memory poisoning

Persistent records that try to become future system rules or trusted facts.

Exfiltration attempts

Output or tool arguments that carry secrets, tokens, credentials, or private data.

Tool and rendering boundaries

Shells, file uploads, email sends, browser rendering, and backend MCP tools.

Harness architecture

A control layer between agents, data, tools, and audit systems.

AIegis is designed as a boundary harness. It does not ask the main agent to safely reason over raw hostile content. It inspects and labels external inputs first, evaluates proposed tool calls against policy, inspects outbound text, and records decisions.

Instruction/data separation

External content is treated as data, not authority. Findings identify hidden text, prompt-like language, suspicious headers, and risky document content before it enters trusted context.

Policy decisions before action

Tool calls are evaluated before execution. AIegis returns allow, block, or approval-required decisions with redacted evidence.

MCP mediation

Run AIegis as an MCP stdio server or place the MCP stdio proxy in front of backend tools so guard decisions happen at the tool boundary.

Auditable outcomes

JSONL audit records are minimized by default and sealed with a hash chain. Eventloom-compatible metadata logs are supported.

Guard surfaces

Practical checks for the places agents actually get hurt.

HTML guard

Detects hidden prompt-like content, suspicious links, and risky markup.

Email guard

Parses RFC-style email, checks headers, inspects HTML parts, and quarantines attachments.

Memory guard

Flags memory poisoning and durable-instruction attempts before retrieval or reuse.

Document guard

Inspects text-like documents and quarantines binary/PDF inputs without trusted extraction.

Tool firewall

Blocks or queues sensitive tools such as shell, deletes, external email, and uploads.

Browser command planning

Builds hardened render command plans while requiring the deployment to provide the sandbox.

Egress guard

Inspects outbound text for secret-like material and returns redacted output.

Approval queue

Records approval-required actions with sensitive arguments redacted for human review.

Quick start

Run it as a CLI, MCP server, proxy, or container sidecar.

AIegis is a Python package with a small operational surface. Start locally, then wire the same policy and audit controls into an agent runtime. 

python -m pip install -e '.[dev]'
printf '<p>Visible</p>' | aiegis inspect-html
aiegis mcp-stdio \
  --policy-file examples/policies.yaml \
  --audit-log .aiegis/audit.jsonl
aiegis mcp-proxy-stdio \
  --approval-log .aiegis/approvals.jsonl \
  -- python backend_mcp_server.py
docker compose -f deploy/docker-compose.yml up --build
